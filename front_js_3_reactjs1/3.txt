public 폴더 설명 : favicon, index, manifest(여기서는 볼 필요 없음)

src 폴더 설명 : js, css 등이 있음. 
    index.js에서, 3행, 5행 지우고 시작.
    index.js, app.js를 제외하고는 모두 지운다.

app.js 내용을 초기화한다.
index.js 내용을 초기화한다.

localhost:3000 에 가본다. 전부 비어있다.

App.js에서 Hello React를 적어보고 저장하면 바로 refresh하여 반영되는 것을 확인해본다.

F12에서 HTML을 확인한다. div태그안에 id="root"가 추가되어 있음을 확인한다.
이는 내가 거기에 쓰는 모든 요소를 생성한다는 것이다.
React는 JS와 함께 컨텐츠(index.html의 div id="root" 태그 안에 들어갈 태그들)를
만들고 HTML로 밀어넣는다. 그래서 HTML이 보이지 않는 것이다.
index.js에서는 ReactDOM.render() 명령어를 볼 수 있는데 이는 App을 통해서 앱을 만드는 명령어이다.
(App.js에서 이 root를 potato로 바꾸면, index.js에서도 potato로 바꿔줘야 에러가 안뜬다. 즉, root는 그냥 이름)

이것이 JS를 통해 HTML을 구성하는 One page application 방식이며 이것을 하게 하는 프레임워크를 SPA라고 부른다.
